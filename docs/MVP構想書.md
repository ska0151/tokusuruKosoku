# MVP（Minimum Viable Product）構想書

**作成日**: 2026-05-04  
**対象範囲**: フェーズ 0～M2 のプロトタイプ～地図UI統合段階  
**目標**: 首都圏 IC 料金比較の**コア機能検証**と**ユーザーフィードバック収集**

---

## 1. MVP の位置付け

### 1.1 基本設計書との関係図

```
┌─────────────────────────────────────────────────────────┐
│ 基本設計書（BDD）: 完全仕様                                  │
│ - フル機能（5IC比較、リアルタイム渋滞、カーナビ連携等）   │
│ - 開発期間: 約 3 ヶ月                                     │
└─────────────────────────────────────────────────────────┘
                       ↑ 段階的拡張
                       │
┌─────────────────────────────────────────────────────────┐
│ MVP: 最小限機能＆ユーザーフィードバック検証                │
│ 開発期間: 4～6 週間                                      │
│ ・ターゲット: 首都圏高速ユーザー（50～100人）             │
│ ・目的: 仮説検証＆市場ニーズ確認                          │
└─────────────────────────────────────────────────────────┘
```

### 1.2 MVP のミッション

**検証すべき 3 つの仮説**

| # | 仮説 | 検証方法 | 成功基準 |
|---|---|---|---|
| **H1** | ドライバーは「料金 ÷ 時間」を比較値として理解・活用できるか | ユーザーテスト（クリックスループ + 実機） | 3人以上が「使えそう」と回答 |
| **H2** | 複数IC自動検出より「お気に入り IC 登録」でもトレンドカバー可能か | 簡易 UI で手入力＋保存機能テスト | 登録 IC 3 件以上で十分か確認 |
| **H3** | 固定所要時間（渋滞情報なし）でも意思決定に足りるか | 実走行テスト 5～10ケース | ±10分誤差以内か判定 |

## 2. MVP スコープ定義

### 2.1 含める機能 ✅

#### 画面

| # | 画面名 | 主要機能 | 備考 |
|---|---|---|---|
| **1** | ホーム | 出発地・目的地入力、最近検索履歴表示 | 最小限 |
| **2** | IC 選択（簡易版） | 手入力で入口 IC / 出口 IC 指定 | 自動検出は不要 |
| **3** | ルート比較 | 3 案表示（最速・バランス・最安） | 差額＋時間差を視覚化 |
| **4** | ルート詳細 | セグメント別料金、割引ルール表示 | 理解度確認用 |

#### 機能

| 機能 | 詳細 | 優先度 |
|---|---|---|
| **ルート計算** | OSRM で 3 ルート並列計算 | 🔴 必須 |
| **料金計算** | 基本料金＋ETC割引（時間帯別）適用 | 🔴 必須 |
| **結果キャッシュ** | 同一 IC ペアの 1 日内再検索は即座返却 | 🟡 後付可 |
| **地図表示** | Leaflet + OSM（ルート可視化） | 🔴 必須 |
| **割引ルール判定** | 平日昼間 / 深夜割引など基本ルール（3 種類） | 🟡 後付可 |
| **外部ナビ連携** | Google Maps / Apple Maps 起動 URI | 🟡 後付可 |

### 2.2 除外する機能 ❌

| 機能 | 理由 | 後日対応 |
|---|---|---|
| **リアルタイム渋滞情報** | NEXCO API の組み込み複雑、固定所要時間で検証 | M2.5 以降 |
| **周辺 IC 自動検出** | 手入力でも可能、UI 複雑さ軽減 | Phase 2 |
| **複数ルート案（5 件以上）** | 3 案に絞って判断性向上 | Phase 2 |
| **ユーザー認証** | 匿名利用で十分、初期段階は不要 | Phase 2 |
| **検索履歴ローカル永続化以外** | クラウド同期は後付 | Phase 2 |
| **高度な割引ルール** | 複雑な通勤割引・企業割引は後付 | Phase 3 |
| **A/B テスト基盤** | Segment / Mixpanel 統合は後付 | Phase 2 |

## 3. MVP 技術スタック（簡約版）

### 3.1 フロントエンド

```
Next.js 14 (TypeScript)
├ pages/
│  ├ index.tsx              // ホーム（出発地・目的地入力）
│  ├ ic-select.tsx          // IC 選択
│  └ routes/[searchId].tsx  // ルート比較
├ components/
│  ├ RouteComparison.tsx    // 3案表示
│  ├ MapDisplay.tsx         // Leaflet 埋め込み
│  └ (その他UI部品)
└ lib/
   ├ api-client.ts         // バックエンド呼び出し
   └ utils.ts              // フォーマッタ等

依存パッケージ:
- react 18
- tailwindcss（UI）
- leaflet + react-leaflet（地図）
- axios（HTTP）
- zustand（状態管理）
```

### 3.2 バックエンド

```
Node.js + Express (TypeScript) — 推奨 理由: 迅速プロトタイプ

src/
├ routes/
│  ├ search.ts             // POST /api/search → OSRM + 料金計算
│  └ ics.ts                // GET /api/ics （手入力値の検証）
├ services/
│  ├ osrmService.ts        // OSRM ルート計算
│  ├ tollService.ts        // 料金テーブル照会＆割引適用
│  └ cacheService.ts       // Redis （簡易キャッシュ）
├ models/
│  └ toll.model.ts         // 料金セグメント定義
└ app.ts

依存パッケージ:
- express
- node-fetch または axios（OSRM API呼び出し）
- ioredis（Redis キャッシュ）
- pg（PostgreSQL）
```

### 3.3 データベース

```sql
-- 最小限スキーマ（手入力 IC 用）
CREATE TABLE ic_masters (
  id VARCHAR(20) PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  lat DECIMAL(10, 8),
  lng DECIMAL(11, 8),
  operator VARCHAR(50)
);

-- 料金マスタ（固定値）
CREATE TABLE toll_segments (
  id SERIAL PRIMARY KEY,
  from_ic_id VARCHAR(20),
  to_ic_id VARCHAR(20),
  distance DECIMAL(8, 2),
  base_toll INT,
  FOREIGN KEY (from_ic_id) REFERENCES ic_masters(id),
  FOREIGN KEY (to_ic_id) REFERENCES ic_masters(id)
);

-- 割引ルール（3種類程度）
CREATE TABLE discount_rules (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100),
  discount_percent DECIMAL(5, 2),
  condition_description VARCHAR(200)
);
```

## 5. データ準備計画

### 5.1 IC マスタデータ

**出所**
- 首都圏 IC リスト：https://www.driveplaza.com （NEXCO）から手作業抽出
- 座標：Google Maps で逆引き or OpenStreetMap Nominatim

**最小限範囲**
- 首都高速: 約 30 箇所
- 東名阪速、中央道、外環等重要ルート: 約 20 箇所
- **計 50 IC から開始**（段階拡張予定）

**格納方法**
```csv
# ics.csv
id, name, lat, lng, operator
IC001, 渋谷出入口, 35.6595, 139.7004, NEXCO-EAST
IC002, 品川出入口, 35.6280, 139.7391, NEXCO-EAST
...
```

### 5.2 料金テーブル

**出所**
- NEXCO 公式サイト + DriveWplaza
- https://www.driveplaza.com/fee/fees.html （距離制・区間定額対象区間）

**作成方式**
1. **初期版** (Week 2): 手入力で 20～30 区間パターン
2. **拡張版** (Phase 2): スクレイピング自動化

**初期データ例**
```csv
from_ic, to_ic, distance, base_toll
IC001, IC101, 45.3, 1200
IC001, IC102, 63.4, 1650
IC001, IC201, 412.5, 7100
```

### 5.3 割引ルール（MVP 3 種類）

| ルール | 条件 | 割引率 |
|---|---|---|
| **ETC 昼間 30%** | 平日 6:00～22:00 | -30% |
| **ETC 夜間 50%** | 平日 22:00～6:00 ＋ 土日祝 | -50% |
| **通勤割引** | 平日 5:30～8:30 / 18:00～20:00 | -30%（別カウント） |
